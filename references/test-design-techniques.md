# Test Design Techniques

## Equivalence Partitioning (等价类划分)

将输入数据划分为若干等价类，每个等价类中的数据对揭露程序中的错误等效。

### 有效性等价类
- 有效输入范围
- 符合格式的数据
- 正常业务数据

### 无效性等价类
- 无效输入范围
- 不符合格式的数据
- 异常业务数据

**示例**: 年龄字段(1-100)
- 有效等价类: 1-100
- 无效等价类: <1, >100, 非数字, 空, 特殊字符

## Boundary Value Analysis (边界值分析)

在等价类边界上和边界附近选择测试数据。

### 标准边界值
- 最小值 (Min)
- 略高于最小值 (Min+)
- 正常值 (Nom)
- 略低于最大值 (Max-)
- 最大值 (Max)

### 健壮性边界值
- 略低于最小值 (Min-)
- 最小值 (Min)
- 略高于最小值 (Min+)
- 正常值 (Nom)
- 略低于最大值 (Max-)
- 最大值 (Max)
- 略高于最大值 (Max+)

## Decision Table Testing (判定表测试)

处理多条件组合的逻辑。

### 判定表结构
| 条件 | 规则1 | 规则2 | 规则3 | ... |
|------|-------|-------|-------|-----|
| 条件1 | Y | Y | N | ... |
| 条件2 | Y | N | - | ... |
| 动作 | 动作1 | 动作2 | 动作3 | ... |

### 示例: 订单折扣
- VIP用户 + 金额>1000 → 20%折扣
- VIP用户 + 金额≤1000 → 10%折扣
- 普通用户 + 金额>1000 → 5%折扣
- 普通用户 + 金额≤1000 → 无折扣

## State Transition Testing (状态迁移测试)

适用于有明确状态流转的系统。

### 状态迁移矩阵
```
         → 待支付  已支付  已发货  已完成  已取消
待支付      -      ✓      ✗      ✗      ✓
已支付      ✗      -      ✓      ✗      ✓
已发货      ✗      ✗      -      ✓      ✗
已完成      ✗      ✗      ✗      -      ✗
已取消      ✗      ✗      ✗      ✗      -
```

### 测试场景
1. 有效状态转换
2. 无效状态转换
3. 循环状态转换
4. 终态测试

## Use Case Testing (用例测试)

基于用户场景设计测试。

### 用例元素
- Actor (参与者)
- Precondition (前置条件)
- Main Flow (主流程)
- Alternative Flow (备选流程)
- Exception Flow (异常流程)
- Postcondition (后置条件)

## Error Guessing (错误推测)

基于经验推测可能的错误。

### 常见错误类型
1. 输入错误
   - 空值
   - 超长输入
   - 特殊字符
   - SQL注入
   - XSS攻击

2. 业务逻辑错误
   - 边界条件
   - 并发冲突
   - 数据一致性
   - 权限绕过

3. 系统错误
   - 超时
   - 内存溢出
   - 磁盘满
   - 网络中断

4. 接口错误
   - 参数缺失
   - 类型不匹配
   - 格式错误
   - 认证失败

## Risk-Based Testing (基于风险的测试)

根据风险优先级分配测试资源。

### 风险矩阵
| | 低影响 | 中影响 | 高影响 |
|---|--------|--------|--------|
| **高概率** | 中风险 | 高风险 | 极高风险 |
| **中概率** | 低风险 | 中风险 | 高风险 |
| **低概率** | 极低风险 | 低风险 | 中风险 |

### 测试优先级
- 极高风险 → 完整测试 + 自动化
- 高风险 → 完整测试
- 中风险 → 重点测试
- 低风险 → 抽查测试

## Pairwise Testing (成对测试)

减少多参数组合测试数量。

### 示例
参数: 浏览器(3种) × 操作系统(3种) × 语言(3种) = 27种组合
使用Pairwise可减少到9种组合。

### 工具
- PICT (Microsoft)
- AllPairs
- Hexawise

## Test Coverage Types

### 功能覆盖
- 需求覆盖: 每个需求点有对应用例
- 功能点覆盖: 每个功能有测试
- 菜单覆盖: 每个菜单项可访问

### 代码覆盖
- 语句覆盖
- 分支覆盖
- 路径覆盖
- 条件覆盖

### 数据覆盖
- 数据类型覆盖
- 数据范围覆盖
- 数据关系覆盖
